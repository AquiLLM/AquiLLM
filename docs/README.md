# Documentation Map

Last updated: 2026-03-25

## Primary folders

- `docs/roadmap/`: delivery status, priorities, and execution plans.
  Plans are organized under `docs/roadmap/plans/{active,pending,completed,superseded}`.
- `docs/specs/`: design specifications and architecture proposals.
- `docs/documents/`: reference documents (architecture, standards, and supporting artifacts).

## Start here

- Current program status: `docs/roadmap/roadmap-status.md`
- Roadmap guide: `docs/roadmap/README.md`
- Specs index: `docs/specs/README.md`
- Documents index: `docs/documents/README.md`

## Naming conventions

- Plans: `YYYY-MM-DD-<topic>.md`
- Specs: `YYYY-MM-DD-<topic>-design.md` (or equivalent design naming)
- Execution records: include `execution-notes`, `rollout-notes`, `handoff`, or `checklist` in filename
